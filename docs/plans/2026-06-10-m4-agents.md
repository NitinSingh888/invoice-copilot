# Milestone 4 тАФ LLM / Agent Layer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. TDD, `- [ ]` steps, each task green (`pytest`+`ruff`+`mypy src`) and committed. Commits author as NitinSingh888 тАФ no Co-Authored-By trailer.

**Goal:** Add the intelligent layer: a provider-agnostic `LLMClient` with a **deterministic `MockClient`** (powers tests + zero-key demo), a real **AnthropicClient** (vision extraction + chat) and a secondary **OpenAIClient**, an ordered **failover** + factory; plus three agents тАФ **extraction** (document тЖТ fields + confidence band), **conversation** (intent тЖТ service calls тЖТ narrated reply), **induction** (corrections тЖТ human-readable rule reasoning) тАФ wired behind `/api/v1/chat` and a file-upload extraction endpoint.

**Architecture:** `clients/llm/*` (pure I/O to providers, swappable; **the app, not the model, executes actions**) тЖТ `agents/*` (use a client + call M2/M3 services; the conversation agent maps intent to deterministic service calls тАФ the LLM proposes, code executes) тЖТ API (`chat`, `invoices/upload`). The **deterministic guard and all money decisions stay in the domain/services** тАФ the LLM never authorizes anything. MockClient makes the whole system run with **no API keys**.

**Tech Stack:** `anthropic`, `openai` SDKs; `pypdf` (extract embedded text), `pillow` (rasterize a page for vision тАФ optional). All real-provider calls are isolated in adapters; tests use MockClient or monkeypatched SDK calls.

**Spec:** ┬з2 (conversation/demo beats), ┬з6.3 (LLM proposes/code decides), ┬з9 (conversation + tools), ┬з10 (model adapter: mock + Anthropic solid, OpenAI secondary), ┬з11 (generalizing extraction + confidence-gated).

### Task 1: LLMClient protocol + types + MockClient
**Files:** `clients/__init__.py`, `clients/llm/__init__.py`, `base.py`, `types.py`, `mock_client.py`; tests `tests/unit/clients/test_mock_client.py`.

- [ ] `types.py` (frozen dataclasses):
  - `ExtractedField(value: str | None, confidence: str)` (confidence in HIGH/MED/LOW).
  - `ExtractedInvoice(vendor, amount, po_number, invoice_number, fields: dict[str, ExtractedField], overall_confidence: str)` тАФ `amount` as `Decimal | None`.
  - `ChatMessage(role: str, content: str)`.
  - `AgentReply(text: str, intent: str, args: dict[str, Any])` тАФ `intent` тИИ {process_batch, explain, approve, route, hold, propose_rule, show_trail, smalltalk}; `args` carries ids etc.
- [ ] `base.py`: `class LLMClient(Protocol)` with `def extract_invoice(self, *, text: str, image_b64: str | None) -> ExtractedInvoice` and `def converse(self, *, history: list[ChatMessage], context: dict[str, Any]) -> AgentReply` and `def explain_rule(self, *, vendor: str, over_pcts: list[Decimal], threshold_pct: Decimal, route: str) -> str` and a `name: str` attribute.
- [ ] `mock_client.py`: `class MockClient` implementing the protocol DETERMINISTICALLY:
  - `extract_invoice`: parse the provided `text` for simple `key: value` lines (vendor/amount/po/invoice_number) the seed/demo fixtures use; default confidence HIGH unless a field is missing/garbled тЖТ MED/LOW. (No randomness.)
  - `converse`: rule-based intent routing mirroring the prototype (regex on the last user message): "process|today" тЖТ process_batch; "why|trail|escalat|injection|cyberdyne" тЖТ explain/show_trail; "approve" тЖТ approve; "route" тЖТ route; "rule|learn" тЖТ propose_rule; else smalltalk. Return a templated `AgentReply`.
  - `explain_rule`: templated string: f"Inferred from corrections at {тАж}% over PO тЖТ generalizes to ~{threshold}%; routes to {route}."
  - `name = "mock"`.
- [ ] Tests: `extract_invoice(text="vendor: Acme Corp\namount: 9800\npo: PO-1\ninvoice_number: A-1")` тЖТ vendor "Acme Corp", amount Decimal("9800"), confidence HIGH; a missing amount тЖТ overall MED/LOW; `converse` intent routing for each keyword; `explain_rule` contains the threshold. All deterministic (call twice тЖТ identical).
- [ ] Commit: `feat(llm): LLMClient protocol + types + deterministic MockClient`.
